# Action Authorization 도메인

## 목적
이 최상위 도메인은 애플리케이션 내에서 특정 액션의 무결성과 정당성을 검증하는 역할을 담당합니다. 이는 정의된 비즈니스 규칙과 데이터 불변성에 따라 수행되는 모든 액션이 올바른지 확인합니다. 이는 일반적으로 초기 접근 제어(요청 권한 부여)가 완료된 후에 이루어집니다.

## 책임
- **비즈니스 규칙 시행**: 사전 정의된 비즈니스 로직(예: 컴포넌트-장치 호환성)에 따라 액션의 내용과 컨텍스트를 검증합니다.
- **데이터 불변성 검사**: 액션이 시스템 데이터의 무결성과 일관성을 유지하는지 확인합니다.
- **하위 검증 오케스트레이션**: 핵심 서비스에 의해 처리되기 전에 다양한 특정 검증 단위들을 활용하고 조합하여 검증 과정을 수행합니다.

## 구조
- `validators/`: 다양한 유형의 액션 또는 데이터 흐름에 대한 특정 검증 로직을 포함합니다.
- `providers/`: (`inter_domain/action_authorization/`에 위치) 이 도메인의 권한 로직과 상호작용하기 위한 안정적인 인터페이스를 다른 도메인에 제공합니다.
