# 텔레메트리 수집 정책

## 목적
이 정책은 들어오는 텔레메트리 데이터 메시지에 대한 전체 비즈니스 규칙을 시행하기 위해 일련의 세분화된 검증기들을 오케스트레이션합니다. 텔레메트리가 수집되기 전에 데이터 무결성 및 시스템 보안을 보장하기 위해 개별 검증 검사가 수행되는 순서와 조건을 정의합니다.

## 책임
- **오케스트레이션**: 정의된 순서로 개별 독립 검증기(예: HMAC, 장치 식별, 시스템 지원, 장치 연결, 청사진 호환성)를 호출합니다.
- **데이터 사전 가져오기/최적화**: 여러 후속 검증기에서 필요한 공통 데이터(예: 장치 및 지원 컴포넌트 객체)를 정책 수준에서 효율적으로 가져와 중복 데이터베이스 조회를 최소화합니다.
- **조기 종료**: 모든 검증 단계가 실패하면 정책 실행을 중지하고 실패를 보고하는 단락(short-circuiting) 로직을 구현합니다.

## 흐름
1. HMAC 검증
2. 장치 식별 검증
3. 시스템 지원 검증
4. 장치 및 지원 컴포넌트 객체 가져오기 (후속 검사를 위한 최적화)
5. 장치 연결 검증
6. 청사진 호환성 검증

## 내용
- `telemetry_ingestion_policy.py`: 이 오케스트레이션 로직을 구현하는 `TelemetryIngestionPolicy` 클래스를 포함합니다.
