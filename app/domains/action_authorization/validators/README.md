# Validators (검증기) 서브도메인

## 목적
이 `authorization` 도메인 내의 서브도메인은 다양한 들어오는 데이터나 요청에 대한 특정 검증 로직을 담고 있습니다. 각 검증기는 핵심 서비스에 의해 처리되기 전에 데이터/요청의 무결성, 진위성 및 규정 준수를 확인하기 위한 일련의 규칙을 캡슐화합니다.

## 책임
- **규칙 시행**: 특정 비즈니스 규칙 및 데이터 무결성 검사를 구현하고 시행합니다.
- **신호 생성**: 설명적인 오류 메시지와 함께 명확한 성공/실패 신호를 반환합니다.
- **의존성 오케스트레이션**: 데이터 수정 자체를 수행하지 않고, 검증에 필요한 데이터를 수집하기 위해 다른 서비스(예: `user_identity`, `device_management`) 호출을 오케스트레이션합니다.

## 내용
- `telemetry/telemetry_ingestion_validator.py`: HMAC 검증, 장치 신원 확인, 하드웨어 청사진과의 부품 호환성 등 들어오는 텔레메트리 데이터를 검증합니다.
