# 사용자 존재 여부 검증기 (User Existence Validator)

## 목적
이 검증기는 주어진 이메일 주소를 가진 사용자가 시스템에 이미 등록되어 있는지 확인하는 단일 책임을 가집니다.

## 검증 로직
1.  `user_identity_providers`를 통해 특정 이메일 주소를 가진 사용자를 데이터베이스에서 조회합니다.
2.  사용자가 존재하면 `(True, user_object)`를 반환합니다.
3.  사용자가 존재하지 않으면 `(False, None)`을 반환합니다.

이 검증기는 '실패' 시 에러 메시지를 반환하는 대신, `Policy` 계층이 분기 처리를 할 수 있도록 상태와 데이터를 반환합니다.
