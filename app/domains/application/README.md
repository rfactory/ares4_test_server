# Application (오케스트레이션) 계층 설계

## 1. 핵심 원칙: 클라이언트별 BFF (Backend-for-Frontend)

이 애플리케이션 계층은 클라이언트 애플리케이션 각각에 전용 오케스트레이션 도메인을 제공하는 BFF(Backend-for-Frontend) 역할을 합니다. 주요 클라이언트는 다음과 같습니다:

-   **관리자 패널 (웹):** 관리자와 조직 관리자를 위한 React Admin 애플리케이션입니다.
-   **Flutter 앱 (모바일):** 모든 사용자 유형(개인, 조직 구성원, 관리자)을 위한 모바일 애플리케이션입니다.

이 패턴은 백엔드가 각 클라이언트의 특정 요구사항에 맞춰 최적화되고 통합된 데이터를 제공하여, 프론트엔드 개발을 단순화합니다.

---

## 2. 디렉토리 구조

"클라이언트별 BFF" 패턴을 반영하기 위해, `application` 디렉토리는 다음과 같이 구조화됩니다:

-   `application/`
    -   `admin_panel/`: React Admin 웹 패널을 위한 오케스트레이터.
        -   `common/`: 모든 관리자 패널 사용자에게 **공통**으로 제공되는 기능(예: 로그인, 내 정보 수정)을 위한 오케스트레이터.
        -   `admin/`: **시스템 관리자 전용** 기능(예: 전체 사용자 관리, 시스템 설정)을 위한 오케스트레이터.
        -   `organization/`: **조직 사용자 전용** 기능(예: 소속 직원 관리, 기업 정보 수정)을 위한 오케스트레이터.
    -   `flutter_app/`: Flutter 모바일 앱을 위한 오케스트레이터.

이 계층 내의 각 하위 디렉토리는 해당 클라이언트 및/또는 역할에 대한 특정 워크플로우를 처리하기 위한 자체 `api` 및 `services` (오케스트레이터/코디네이터용)를 포함합니다.
